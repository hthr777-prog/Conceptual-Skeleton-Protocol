# 🧠 CSP (Conceptual Skeleton Protocol) — מדריך למשתמש

ברוכים הבאים למאגר הכלים הרשמי של פרוטוקול **CSP**. פרוטוקול זה מהווה "משטר חשיבה פנימי" למודלים של בינה מלאכותית, שנועד לייצוב משמעות ומניעת בלבול מושגי.

## 🚀 תחילת עבודה (הגדרה ב-Gemini Gems)

כדי להטמיע את הפרוטוקול בתוך Gem ייעודי, בצעו את השלבים הבאים:

1. **הורדה:** הורידו את קובץ ה-CSP (פורמט PDF/Text) מהקישור שקיבלתם.
2. **יצירת Gem:** פתחו את ממשק ה-Gems ב-Gemini ולחצו על ה-+ (**הוספת קובץ / ידע**).
3. **ייבוא:** העלו את קובץ ה-CSP למערכת.
4. 
**הגדרה:** ודאו בהנחיות המערכת (System Instructions) שה-Gem מונחה להשתמש בקובץ זה כ**תשתית הניתוח המרכזית** לכל פלט.



---

## 🛠️ שילוב עם הוראות נוספות

ניתן להשתמש ב-CSP כבסיס שעליו נבנות שכבות ידע נוספות. אם ה-Gem שלכם כולל הנחיות ספציפיות לתחום מסוים:

* **הכלל:** CSP משמש כתשתית הניתוח; ההנחיות האחרות נבנות עליו ולא מחליפות אותו.


* **הנחיה מומלצת:**
> "השתמש ב-CSP כבסיס לכל ניתוח מושגי." 
> 
> 



---

## 🔍 התאמת עומק התשובה

כברירת מחדל, CSP שואף למינימליזם ודיוק (שימוש ברכיבי היסוד F ו-C בלבד). אם נדרש ניתוח נרחב יותר, ניתן להשתמש בפקודות הבאות:

* 
**לניתוח מקיף:** "השתמש ב-CSP אך אל תגביל את מספר המשמעויות או הרכיבים (השתמש גם ברכיבי הנספח במידת הצורך)".


* 
**לפירוט מבני:** "בצע ניתוח CSP מלא והצג פירוט לפי כל רכיבי הפרוטוקול ששימשו בתהליך".

## 💡 טיפים לשימוש מיטבי

* 
**מיקוד:** שאלו שאלה אחת בכל פעם כדי לאפשר למודל לבצע "קריאה מבנית" (Stage 0) מדויקת.


* 
**טיפול בעמימות:** אם נתקלתם במושג בעל מספר פרשנויות, בקשו במפורש: "בצע הפרדת משמעויות (Disambiguation) לפני המענה".


* **שאלות נורמטיביות:** בשאלות של "מה עדיף?" או "מותר/אסור", CSP יעזור לכם למפות את הספקטרום ואת "מטריצת הגבולות" (Boundary Matrix) במקום להחליט עבורכם.
