# CSP — Conceptual Skeleton Protocol (v1.0)
### Stop Summarizing. Start Understanding.

**CSP** is a minimal, logic-based protocol designed to eliminate conceptual confusion (equivocation) in human-AI interaction and structured thought.

Most "hallucinations" or vague answers don't stem from a lack of data, but from a failure to separate different meanings of the same term. CSP forces a structural breakdown before a single word of explanation is written.

---

## The Problem: Conceptual Noise
When we ask "What is Time?", we often get a "grocery list" answer that mixes physics, psychology, and math. This mixture creates noise. 

**Equivocation Example:**
* **Time** = Atomic oscillations (Physics) 
* **Time** = Sense of urgency (Phenomenology)
* **Time** = A mathematical axis (Geometry)

Mixing these categories results in a redundant, blurry response.

---

## The Protocol: Two Steps to Clarity

### 1. Disambiguation (Select Type)
Before explaining, you must choose **one** conceptual path. You cannot explain "Time" in general; you must choose to explain "Physical Time" OR "Subjective Time".

### 2. Extraction (The Skeleton)
Once the type is selected, it is decomposed into its **Primitives**:

* **Foundation (F):** What exists? (Ontology / Pure Structure).
* **Experience (E):** How is it perceived? (Phenomenology / Subjective).
* **Causality (C):** Why does it exist? (Cause, Purpose, and Trade-offs).

---

## Principles

1.  **Type Purity:** No category mixing. If you chose "Physical", don't slip into "Emotional".
2.  **Occam’s Razor:** Only what is required for structural understanding. No history, no fluff.
3.  **Ontology ≠ Phenomenology:** Never repeat the same idea in different words. If it’s a physical fact, it belongs in **F**. If it’s a feeling, it belongs in **E**.

---

## How to use with LLMs (ChatGPT, Claude, Gemini)

The fastest way to implement CSP is to provide the model with the protocol as a system prompt or a starting message:

> "Analyze the following concept using the **CSP Protocol**:
> 1. Identify possible types.
> 2. Wait for my selection.
> 3. Extract the Skeleton (F, E, C) for the chosen type only."
